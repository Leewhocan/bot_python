# Moscow Zoo Totem Bot

Телеграм-бот для интерактивной викторины «Тотемное животное Московского зоопарка».  
Помогает подобрать животное‑покровителя и рассказывает о программе опеки «Клуб друзей зоопарка».

## Что умеет бот

- Запускается по команде `/start` (или нажатием кнопки «Попробовать ещё раз»).
- Показывает приветственное сообщение с логотипом и случайной фразой.
- Проводит интерактивную викторину с inline‑кнопками (один вариант ответа — одно нажатие).
- Подсчитывает результат по системе баллов (веса ответов).
- Показывает итоговое животное с персонализированным описанием.
- Отправляет изображение животного (если файл добавлен в папку `assets/images/`).
- Рассказывает о программе опеки «Клуб друзей зоопарка» и даёт ссылку на официальную страницу.
- Позволяет поделиться результатом (кнопка формирует текст со встроенной ссылкой на бота).
- Даёт возможность связаться с сотрудником зоопарка – результат викторины пересылается сотруднику (и администратору).
- Собирает отзывы пользователей (сохраняются в SQLite, администратор сразу получает уведомление).
- Администратор может посмотреть последние отзывы командой `/feedbacks` и статистику по викторине командой `/stats`.
- Позволяет перезапустить викторину в любой момент через постоянную кнопку на клавиатуре («Попробовать ещё раз»).

## Дополнительные возможности

- Случайная приветственная фраза при каждом `/start`.
- Приветственное изображение‑логотип (`assets/images/logo.png`).
- Автоматические уведомления администратору:
  - о каждом завершении викторины (с результатом),
  - о каждом новом отзыве,
  - о запросе связи с сотрудником.
- Команда администратора `/stats` – показывает общее количество прохождений, популярных животных и последние результаты.
- Все чувствительные настройки (токен, ID администратора и сотрудников) вынесены в `.env`.
- Полная асинхронность, готовая к высоким нагрузкам.

## Технологии

- Python 3.11+
- aiogram 3
- SQLite (aiosqlite)
- python‑dotenv

## Установка

1. **Клонируйте репозиторий или распакуйте архив.**

2. **Создайте виртуальное окружение:**

   ```bash
   python -m venv .venv
   ```

3. **Активируйте его:**
   - Windows (PowerShell / CMD): `.venv\Scripts\activate`
   - Git Bash на Windows: `source .venv/Scripts/activate`
   - Linux/macOS: `source .venv/bin/activate`

4. **Установите зависимости:**

   ```bash
   pip install -r requirements.txt
   ```

5. **Создайте файл `.env` в корне проекта** со следующим содержимым:

   ```env
   BOT_TOKEN=ваш_токен_от_BotFather
   ADMIN_ID=ваш_telegram_id
   STAFF_ID=id_сотрудника_зоопарка
   ```

   `STAFF_ID` может содержать несколько ID через запятую (например, `123456789,987654321`).  
   Если сотрудник не указан, заявки на связь будет получать только администратор.

6. **Добавьте изображения:**
   - логотип `assets/images/logo.png` (любое приветственное изображение);
   - фотографии животных в `assets/images/` с именами `lion.jpg`, `penguin.jpg`, `zebra.jpg`, `flamingo.jpg`, `elephant.jpg`.

   При отсутствии картинок бот отправит только текстовое описание животного.

7. **Запустите бота:**
   ```bash
   python main.py
   ```

## Как работает викторина

Вопросы, варианты ответов и «веса» для каждого животного описаны в файле `data/quiz_data.py`.  
Каждый выбранный вариант начисляет баллы одному или нескольким животным.  
После последнего вопроса бот подсчитывает сумму баллов по всем ответам и выбирает животное с максимальным результатом.

**Пример веса ответа:**

```python
{"text": "Ранний подъём и зарядка", "weights": {"Лев": 2, "Зебра": 1}}
```

Если пользователь выбирает этот вариант, Лев получает 2 балла, Зебра — 1 балл.

## Как добавить новое животное

1. Добавьте запись в словарь `ANIMALS` в `data/quiz_data.py` (описание, путь к картинке, стикер).
2. Положите изображение в `assets/images/`.
3. Во все вопросы, где вы хотите «учитывать» новое животное, добавьте в `weights` соответствующие баллы.
4. Перезапустите бота.

## Масштабируемость

Проект спроектирован с учётом возможного роста нагрузки:

- Асинхронный фреймворк aiogram 3 обеспечивает высокую производительность.
- Состояния викторины изолированы для каждого пользователя (FSM в памяти; легко перенести на Redis).
- Вопросы и описания животных вынесены в отдельный модуль, не затрагивая основную логику.
- Код разделён на независимые модули (handlers, keyboards, database, utils).
- Данные хранятся в SQLite; при необходимости СУБД можно заменить на PostgreSQL без изменения логики.
- Все административные функции защищены проверкой `ADMIN_ID`.

## Конфиденциальность и безопасность

- Бот собирает только те данные, которые необходимы для работы:
  - Telegram ID,
  - username (если указан),
  - результат викторины,
  - текст отзыва (только если пользователь его отправил).
- Никакие дополнительные личные данные не запрашиваются.
- Вся информация хранится локально в базе SQLite.
- Доступ к административным командам (`/feedbacks`, `/stats`) ограничен `ADMIN_ID`.
- Токен и ID администратора/сотрудников хранятся в `.env` и исключены из системы контроля версий (`.gitignore`).

## Уникальность и креативность

- Бот не просто проводит тест, а создаёт игровую атмосферу с дружелюбным тоном и случайными приветствиями.
- Каждому животному присвоен характерный образ и программа опеки (например, «Королевский прайд» для Льва, «Ледяная забота» для Пингвина).
- Результат сопровождается изображением и кратким описанием, мотивирующим принять участие в программе опеки.
- Есть кнопка «Поделиться», которая генерирует готовый текст со встроенной ссылкой на бота – отличный инструмент для вирусного распространения.
- Пользователь может в любой момент перезапустить викторину одной кнопкой.
